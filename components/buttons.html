Usage

Use buttons to trigger actions and links. Buttons can contain a combination of a clear label and an icon while links are always text. Buttons are typically placed throughout your UI — dialogs, modal windows, forms, and toolbars.

Types

Placeholder example component

Primary button (high emphasis)

Primary buttons have more emphasis, as they use a color fill to stand out on the page, and are used as the most important action on the page.

Secondary button (medium emphasis)

Secondary buttons are used for more emphasis than text buttons but are not the most important action on the page.

Text button (low emphasis)

Text buttons are typically used for less important actions.

Toggle button

Toggle buttons group a set of actions together using layout and spacing. They’re used when switching between similar view types of content where an icon represents each options. If you’re looking to use labels, the tab component is likely the correct use.

Hierarchy

Balance obvious, easy, and possible

A layout should contain a single prominent button that makes it clear that other buttons have less importance in the hierarchy. This high-emphasis button commands the most attention.

Other buttons

An app can show more than one button in a layout at a time, so a high-emphasis button can be accompanied by medium- and low-emphasis buttons that perform less important actions. When using multiple buttons, ensure the available state of one button doesn’t look like the disabled state of another.
