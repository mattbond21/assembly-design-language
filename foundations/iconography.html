Icons are visual representations of commands, files, devices, directories, or common actions, and are used to provide visual context to enhance usability. They should be simple, bold enough to grab attention, and memorable. We utilize Material design outlined stroke icons in our set, and create our own if necessary.

Icon sizes

Our icons are displayed at four different sizes — xsmall, small, medium, and large. We only design our icons at one size and then they are resizes by code in production.

Quizlet icon set

Our icon set is available in Figma. If you’re contributing a new icon, please be sure to follow these guidelines to create one, and make a component for it in Figma.

Creating new icons

Grid and layout

Icons should be designed to be contained in a 20pt grid but displayed on 24pt canvas. This allows for even spacing in the UI and uniformity within the icon set.

Keylines and shapes

Create icons on the grid using basic foundational shapes, which include; circles, rectangles (horizontal and vertical), and squares. Sticking to these shapes in the grid keeps consistent proportions throughout the icon set.

Pixel alignment

To avoid distorting an icon, position icons “on pixel” by making the X and Y coordinates into integers, without decimals.
